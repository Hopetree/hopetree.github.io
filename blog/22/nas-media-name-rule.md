# NAS 媒体文件命名规范

玩 NAS 的媒体库，要实现自动刮削元数据、正确识别剧集和电影，并在客户端中显示海报、剧情简介与分集信息，文件夹结构与命名规范是最关键的基础条件。实践中，遵循官方推荐的结构可以显著降低误识别与需要手工修复的概率。

以下内容以《凡人修仙传》等常见资源为例，总结一套在 2025–2026 年仍然稳定有效的命名与目录规范。

## 1. 文件名解析

举个例子，这里有个凡人修仙传的文件名称：

**A.Record.Of.Mortals.Journey.To.Immortality.S01E001.2020.2160p.WEB-DL.H264.AAC-PanWEB**

这个文件名是典型的 **Emby/Plex/Jellyfin** 等媒体服务器使用的命名格式（也常见于BT下载/收藏圈），它包含了非常完整的信息。下面逐段给你拆解：


| 部分                                    | 含义                                                                 | 说明                                                                 |
|-----------------------------------------|----------------------------------------------------------------------|----------------------------------------------------------------------|
| A.Record.Of... | 剧集/动画的英文名称                                                  | 《凡人修仙传》最常用的英文译名（A Record of Mortal's Journey to Immortality）<br>注意点号分隔，这是Emby标准写法，避免空格 |
| S01                                     | 第一季（Season 01）                                                  | 表示这是第1季                                                       |
| E001                                    | 第1集（Episode 001）                                                 | 第一集（很多国漫/国剧用001、002这种三位数编号）                       |
| 2020                                    | 首播/制作年份                                                        | 这部作品的动画版于2020年开始在B站播出（真人剧版反而是后面才出）       |
| 2160p                                   | 视频分辨率                                                           | 也就是4K UHD（3840×2160）                                            |
| WEB-DL                                  | 片源类型                                                             | 从流媒体平台（如腾讯、爱奇艺、B站等）直接下载的原盘资源<br>画质和音质通常最好，属于常见的高质量国剧/国漫片源 |
| H264                                    | 视频编码格式                                                         | 使用H.264/AVC编码（最常见的4K兼容编码，虽然现在H265/HEVC更流行）     |
| AAC                                     | 音频编码格式                                                         | AAC音频（常见于网络剧集/动画，一般是2.0或5.1声道）                   |
| PanWEB                                  | 发布组/字幕组/压制组名称                                             | 国内非常有名的4K压制组，专做高码率WEB-DL 4K资源，质量在国漫圈口碑很好 |

一句话总结：这是 《凡人修仙传》动画版 第一季 第1集 的 4K超清版本，2020年原版WEB源，由PanWEB小组压制，视频用H.264编码，音频AAC，属于目前国内能见到的最高画质版本之一。

## 2. 命名规范

### 2.1. 电视剧 / 动画 / 网剧

**推荐的目录结构**

```
TV根目录（例如“电视剧”或“TV Shows”）
└── 剧名 (首播年份)
    └── Season 01
    │   ├── 剧名.S01E01.mkv
    │   ├── 剧名.S01E02.mkv
    │   └── …
    ├── Season 02
    ├── Specials
    └── extras
```

- 剧名后建议带首播年份，用于区分同名作品  
- 每一季使用 `Season 01`、`Season 02` 这样的标准写法  
- `Specials` 或 `Season 00` 用于番外、OVA、花絮等内容  
- `extras` 用于该剧级别的访谈、预告片等附加视频  

**季目录写法的稳定性对比**

| 写法        | 兼容性 | 实际表现 | 说明 |
|-------------|--------|----------|------|
| Season 01   | 高     | 稳定     | 最推荐 |
| Season 1    | 较高   | 稳定     | 可用 |
| S01         | 中     | 偶有问题 | 不建议长期使用 |
| S1          | 低     | 易失败   | 应避免 |
| 第一季等中文 | 很低   | 基本不识别 | 应避免 |

**剧集文件名**

最基本且稳定的格式：

```
剧名.S01E03.mkv
剧名 - S01E03 - 本集标题.mkv
```

常见的完整信息写法（也可使用中文名）：
```
A.Record.of.Mortals.Journey.to.Immortality.S01E001.2020.2160p.WEB-DL.H264.AAC.mkv
凡人修仙传.S01E001.2020.2160p.WEB-DL.H264.AAC.mkv
```

**多集或分段文件**

- `S01E01-E03`：多集合并在一个文件  
- `S01E01.pt1`、`S01E01.pt2`：单集拆分  

### 2.2. 电影（Movies）

**推荐结构**

```
Movies根目录
└── 电影名 (上映年份)
    ├── 电影名 (年份).BluRay.2160p.REMUX.mkv
    ├── 电影名 (年份) - 1080p x265.mkv
    ├── 电影名 (年份) - Directors Cut.mkv
    ├── extras
    └── trailer.mkv
```

- 电影文件夹必须包含上映年份，使用英文括号  
- 同一电影的多个版本应以“电影名 (年份) - 版本说明”命名  
- Emby 会在播放端提供版本切换  

示例：

```
盗梦空间 (2010) - REMUX 2160p HDR.mkv
盗梦空间 (2010) - 1080p x265 10bit.mkv
盗梦空间 (2010) - Directors Cut.mkv
```

### 2.3. 动漫、国漫与日韩剧的常用做法

**国漫（集数较多）**

```
凡人修仙传 (2020)
└── Season 01
    ├── 凡人修仙传.S01E001.2160p.WEB-DL.mkv
    ├── …
    └── 凡人修仙传.S01E258.mkv
```

如果官方有明确分季，则对应使用 `Season 02`、`Season 03` 等。番外或特别篇放入 `Specials` 或 `Season 00`。

**日漫 / 韩剧**

- 统一使用 `Season 01`、`Season 02`
- OVA、特典放入 `Specials`
- 文件名使用 `番名 - S01E01 - 标题` 或 `番名.S01E01`

## 3. 实际使用中的注意事项

- 年份在重名作品中非常重要  
- 避免在文件名中使用中文冒号、问号、星号等非法字符  
- 目录优先级：`Season XX` 高于 `SXX`，远高于纯数字目录  
- 刮削异常时，优先检查媒体库的元数据源与语言设置  
- 常用的重命名与整理工具包括 FileBot 和 TinyMediaManager

按照以上结构整理后，大多数资源都可以被 Emby 类似的媒体库自动正确识别，减少人工干预，并保持库内显示的一致性。
